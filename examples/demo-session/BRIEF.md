# Brief

## Question

Стоит ли **соло-разработчику** перейти с **SQLite** на **PostgreSQL** для **MVP**?

## Context

Классический trade-off для раннего продукта: SQLite минимизирует операционную сложность (один файл, нет отдельного сервера БД), PostgreSQL даёт конкурентную запись, расширяемость, экосистему (миграции, реплики, managed-хостинг).

Универсальный архитектурный вопрос без привязки к конкретному репозиторию.

## Success criteria

1. Чёткая рекомендация: **остаться на SQLite / перейти сразу / перейти при триггерах** — с обоснованием.
2. Критерии «когда пора» с конкретными порогами.
3. Оценка стоимости перехода для соло.
4. Сводная таблица сравнения вариантов в ANSWER.

## Scope

- Выбор БД для MVP одним разработчиком.
- Типичные стеки (Node, Python, Go, Rails) — общие паттерны.

## Out of scope

- Детальное сравнение MySQL, MongoDB.
- Production-кластер PostgreSQL.
