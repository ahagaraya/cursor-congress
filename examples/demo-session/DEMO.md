# Demo session — SQLite vs PostgreSQL для MVP

Публичный пример **полной** сессии Congress (`commission_mode: full`), прошедший `validate --gate full`.

## Вопрос

Стоит ли соло-разработчику перейти с SQLite на PostgreSQL для MVP?

## Вводные (синтетические)

| Параметр | Значение |
| --- | --- |
| Тип | Сайд-проект |
| Нагрузка | Автор + тестеры |
| Хостинг | Один VPS |
| Данные | Персональные данные пользователей |
| Боли SQLite | Мелкие (миграции, типы) |

## Вердикт (кратко)

**Остаться на SQLite** с обязательным контуром защиты персональных данных; PostgreSQL — при документированных триггерах.
